## Перевірені конфігурації

- Native Ubuntu Linux 22.04 на ПК з характеристиками, аналогічними тестовому ноутбуку/ПК команди: демо запустилося успішно.
- Native Ubuntu Linux 22.04 на ПК з MSI RTX 3090 SUPRIM: демо запустилося успішно.
- Windows 10/11 + WSL2 може запускати headless-частину симуляції, але Gazebo GUI через WSLg може не рендеритися коректно. Для учасників та демонстрацій рекомендовано використовувати саме native Ubuntu 22.04.

## Рекомендована конфігурація

- ОС: Ubuntu Linux 22.04 LTS, встановлена напряму на ПК.
- CPU: 6 фізичних ядер або більше, сучасний Intel Core i5/i7/i9 чи AMD Ryzen 5/7/9.
- RAM: 32 GB.
- GPU: дискретна NVIDIA GTX 1660 / RTX 2060 або краще, з актуальним proprietary NVIDIA driver.
- VRAM: 6 GB або більше.
- Диск: SSD/NVMe, мінімум 80 GB вільного місця після встановлення ОС.
- Інтернет: стабільний доступ для встановлення ROS, Gazebo, PX4, Python-пакетів і Git submodules.
- Монітор: Full HD або більше, бажано не через віддалений робочий стіл.

Це рекомендований рівень для комфортної роботи з Gazebo GUI, одночасного запуску кількох PX4 SITL процесів, редагування коду та швидшої збірки PX4.

## Мінімальна конфігурація

- ОС: Ubuntu Linux 22.04 LTS.
- CPU: 4 фізичні ядра.
- RAM: 16 GB.
- GPU: інтегрована Intel Iris Xe / AMD Vega або дискретна NVIDIA початкового рівня.
- Диск: SSD, 50 GB вільного місця.
- Підтримка OpenGL/GPU acceleration в Linux.

На мінімальній конфігурації демо може запускатися, але можливі фризи Gazebo, нижчий FPS, довша збірка PX4 і повільніший старт симуляції.

## Обов’язкове програмне середовище

- Ubuntu 22.04 LTS.
- Gazebo Harmonic (`gz-harmonic`).
- ROS 2 Humble Desktop (`ros-humble-desktop`).
- ROS dev tools (`ros-dev-tools`).
- PX4 Autopilot v1.15.4.
- Python 3 і pip.
- MAVSDK Python.
- Git з підтримкою submodules.
- Базові пакети для збірки PX4, які ставить `PX4-Autopilot/Tools/setup/ubuntu.sh --no-nuttx`.

Проєкт очікує, що PX4 вже встановлено і зібрано. Типовий шлях:

```bash
~/PX4-Autopilot
```

Якщо PX4 лежить в іншій папці, запускати треба з явно заданим шляхом:

```bash
PX4_DIR=/home/<user>/PX4-Autopilot ./run_linux
```

## Що треба перевірити перед початком

1. Ubuntu саме 22.04:

```bash
lsb_release -a
```

2. Gazebo запускається:

```bash
gz sim
```

3. ROS 2 доступний:

```bash
source /opt/ros/humble/setup.bash
ros2 --help
```

4. PX4 SITL зібраний:

```bash
cd ~/PX4-Autopilot
make px4_sitl gz_x500
```

5. GPU acceleration працює:

```bash
glxinfo -B
```

У виводі бажано бачити реальний GPU, а не `llvmpipe`. Якщо виводиться `llvmpipe`, Gazebo буде працювати через CPU-rendering і може дуже сильно гальмувати.

## Чого краще уникати

- Windows + WSL2 як основне середовище для Gazebo GUI. У наших тестах PX4/Gazebo могли запускатися, але GUI Gazebo не рендерився нормально.
- Віртуальні машини без проброшеного GPU.
- Remote Desktop / VNC без нормального OpenGL acceleration.
- Старі NVIDIA драйвери або відкритий `nouveau` драйвер замість proprietary NVIDIA driver.
- HDD замість SSD.
- RAM 8 GB або менше.
